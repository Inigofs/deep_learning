    def file_load(dataset,animal,size,scaler_range):
    X,y = [],[]
    scaler= MinMaxScaler(feature_range= scaler_range)
    for file in glob.glob('/content/drive/My Drive/MDS/38_Deep Learning/practice/data/catsVSdogs/' + dataset + '/' + animal + '/*.jpg'):
        # get the X features
        image = imread(file)
        X.append(np.array([scaler.fit_transform(elem) for elem in resize(image, size, mode="reflect")]))
        # get the y labels
        if animal == 'cats':
            y.append(0)
        elif animal == 'dogs':
            y.append(1)
        else:
            y.append(99)
    return zip(X,y)

    Model 1¶


  • Diseo de la arquitectura del modelo

    • El modelo consitir en un modelo secuencial (la ejecucin se realiza capa tras capa sin posibilidad de saltos ni de ejecucin simultnea) con las siguientes capas:

  • Capa input.

    • Las imagenes reducidas a 64x64 x3 channels (RGB). Se introduce en la primera capa de convolucin en el parmetro input_shape.

  • Capa convolucin.

    • Transforma la imagn segn el kernel elegido. Cada kernel se ejecuta en todos los channels a la vez dando como resultado una imagen plana, es decir, con un nico channel, por tanto, el output de la convolucin sern una imagen filtrada con tantos channels como kernel ejecutados sobre ella.
    El objetivo del kernel es reconocer una parte de la imagen como puedan ser las lneas rectas verticales, diagonales, caras, etc.

    padding: se opta por valid que reduce el tamao de la foto en funcin de los pxeles que son NA en el kernel. En este caso, como el kernel es de 3x3 se reduce en 2 pxeles por capa de convolucin.

  • Activacin.

    • Realiza la transformacin no-lineal del modelo. Se escoge ReLu para evitar posibles problemas de valores cercanos a cero de la derivada primera en el back-propagation, que no permitira mejorar el modelo en dicha pasada.

  • Capa pooling.

    • Reduce el tamao el tamao de la imagen en funcin de la regin establecida, obteniendo el valor principal de la estadstica seleccionada de dicha regin. Normalmente, suele ser el valor mximo como sea escogido en esta ocasin.

  • Tcnica de dropout.

    • Tcnica que reduce el overfitting, ocasionado por las numerosas capas que generan pesos nuevos, anulando aleatoriamente un nmero preestablecido de neuronas, en esta ocasin 25%. Esta tcnica obliga al modelo a adaptarse, a aprender de distintas situaciones. Es decir, es una manera de generalizar el modelo y no permitirle que se aprenda el training set ya que al no contar con las mismas neuronas en cada poca y tener que seguir clasificando bien la neuronas no se especializan tanto no pudiendo aprenderse el training set.

  • Capa flatten.

    • Que combina los distintos channels en uno solo para poder trabajar con capas densas propias de las redes neuronales.

  • Capa densa.

    • Una capa clsica de las redes neuronales. En este caso es la capa output.

  • Activacin.

    • Al igual que se ha mencionado antes, realiza la transformacin no-lineal del modelo.

    Para la capa output y su activacin, habra dos posibilidades:

    1. Capa densa con 2 neuronas + activacin softMax para normalizar a las dos posibles clases.
    2. Capa densa con 1 neurona + activacin sigmoide que define una clase binaria.

    Early Stopping strategy¶

    Se comprueba que la tcnica early stopping detiene la ejecucin del modelo, segn los parmetros definidos, reduciendo el overfitting del modelo.


  • Tcnica Early stopping.

    • Esta tcnica detiene el entrenamiento del modelo cuando se cumplen ciertos criterios establecidos reduciendo la probabilidad de overfitting ya que no se le deja al modelo seguir entrenando, es decir, aprendiendo y ajustndose al training set.
    Los criterios de parada son enfuncin de la variacin de la funcin de prdida, cuando esta no mejora en un valor determinado y/o no se produce una mejora en k pocas consecutivas.
    Para el uso de esta tcnica es preciso utilizar cross-validation para tener un grupo de validacin donde chequear lo mencionado.

    Configuracin:

    • min_delta: 0. Mejora mnima de la funcin de prdida.
    • patience: 3. Nmero de pocas sin la mejora mnima en la funcin de prdida.


  • Cross-Validation.

    • Configuracin: 30%

    Image Augmentation Techniques strategy¶

    Tcnica que aumenta el training set generando nuevos patrones transformados de manera aleatoria.
    Muy usado en data sets pequeos para aumentar el volumen del training set sin incurrir en overfitting del modelo.

    Se compone de los siguientes pasos:

    1. Arquitectura del modelo
    2. Configuracin de la aleatoriedad de la generacin de imgenes [ImageDataGenerator]
    3. Generacin de las imgenes del train set [Train_datagen.flow] y del validation set [Validation_generator]
    4. Entrenamietno del modelo [Fit_generator]

  • Confiuracin.
    • rotacin de imagen. Se define el rango de los grados a rotar [40].
    • traslacin en horizontal y vertical. Se define el rango de traslacin en porcentage [0.2].

    • zoom. Se define el rango del porcentage de zoom a aplicar [0.2].

    Trained models strategy¶

    Esta estrategia consiste en utilizar modelos ya entrenados y comprobados que tienen una arquitectura buena adaptndolos al nuevo dataset de estudio.
    Fases:

      1. Definir el modelo base: modelo entrenado
      1. Adaptar el modelo base al nuevo data set aadiendo las capas necesarias.

        Facores a considerar:

        • 2.1 Dimensin de la capa output
        • 2.2 Dimensin de las imgenes en la capa output
        • 2.3 Nmero de clases a clasificar
      1. Entrenamiento del modelo.
        • 3.1 Entrenamiento de la adpatacin al nuevo data set, las nuevas capas aadidas, congelando capas del modelo base.
        • 3.2 Entrenamiento de todo el modelo (modelo base + adaptacin).

    ResNet50 (1)¶


    1. Arquitectura del modelo.

    • blocks: Nmero de bloques del arquitectura (4 capas densas).
    • include_top: Aplicain de connections residual. Consiste en dar la opcin al modelo de saltarse un bloque. Con esto se consigue por un lado dejar al modelo que optimice las primeras capas (lejos del output) y, por otro, evitar los valores cercanos a cero que se pueden obtener en la fase del back-propagation.
    • weights: Inicializacin de los pesos. Factor importantisimo en los modelos de redes neuronales ya que no son robustos.
      Esta falta de robustez implica que dependiendo de estos valores iniciales se llega a un mnimo local u otro. Para solventar esta eventualidad se corre el modelo varias veces (nmero de pocas).
      Otro factor que ayuda al modelo a encontrar el mnimo global es el momentum solo no es suficiente.
      Por lo general los modelos pre-entrenados dan tres opciones:
      1. Inicializacin aleatoria [random initialization].
      2. Los pesos del modelo pre-entrenado.[imagenet].
      3. Pesos pre-establecidos [path to the weights file to be loaded].

    Best practices sugieren entrenar primeramente con los pesos del modelo pre-entrenado para terminar con un entrenamiento de pocas pocas de todo el modelo.

    • input_tensor: input del data set de estudio que se usa como input del modelo pre-entrenado.
    • input_shape: dimensiones de la imagen -width, heigth & num. of channels (num=3)- (solo cuando include_top = False).
    • pooling: modo de pooling de las caractersticas de la regin (solo cuando include_top = False).
      1. None no se reduce (dimensin del output del modelo: 4D tensor output de la ltima capa de convolucin).
      2. avg se aplica la media para extraer la caracterstica de la regin (dimensin del output del modelo: 2D tensor).
      3. max se aplica el valor mximo para extraer la caracterstica de la regin (dimensin del output del modelo: 2D tensor)
    • classes: Nmero de clases a clasificar. Tiene que ser coherente con el data set en estudio.
